# Changelog

All notable changes to `trans-latin` are recorded here.

## 0.1.0-pre.0 - In progress

### Added

- Auditable translation-record schema and validator.
- Latin translation-unit schema and validator.
- Witness provenance schema and validator.
- Processing handoff schema and validator.
- Audit event schema and validator.
- End-to-end audit-chain validation.
- Audit manifest schema, generator, and validator.
- Review packet exporter.
- Release bundle staging and validation.
- GitHub Actions workflows for full QA, package smoke testing, and release artifacts.
- Python package scaffold under `src/trans_latin`.
- Console entrypoint wrappers for translation record and Latin unit validation.
- Shared translator-spine documentation aligned with `Trans-heb`.
- Language-processing repository boundary documentation.
- Latin transcription and normalization policies.

### Production status

`trans-latin` is not yet a complete language-processing engine. It is a production-shaped scholarly audit and translation repository prepared to interoperate with a future dedicated language-processing repo.
